## The adopted identity (Model C)

*A Map for Mortals* is a **curated work of practical philosophy built on a purposive
discovery corpus**, whose publishable claims each pass through a **claim-specific
confirmation dossier**. It is not a frequency map of human wisdom, and no public
sentence may imply otherwise.

Binding language rules:
1. All recurrence/convergence/saturation statements are **bounded to a declared frame**
   ("recurs in X of Y sampled source families", "across 5 of 6 prompted wave-1
   modules") — never "human wisdom agrees".
2. Editorial voice is **visibly separate** from mapped evidence, on the page.
3. Reader-facing verification language is "text checked to [edition]", never a bare
   "verified".
4. Conceptual diagrams are labelled schematic; evidence plots require structured source
   data.

## The layer discipline (target architecture)

Six layers, no layer silently rewriting the one before: L0 acquisition → L1
source/edition → L2 sourced expression → L3 interpretation → L4 synthesis
(claims/forks/reified relations) → L5 evidence dossier → L6 publication assertion.
The current store is v0.3 (units conflate L2+L3; claims are thin L4). Migration is
**pilot-first and publication-driven** — page 8 proves the model, then the skeleton,
then only units a chapter needs. The 346 discovery units keep their v0.3 shape until
used; they are labelled **discovery corpus v0.3**.

## Gate sequence (from the blueprint, tracked in STATE.md)

- **Gate A — truth model**: schema v0.4 contracts, registries normalised, validators +
  negative fixtures. *(validators: partial — cross-file + linter live; schema contracts
  pending)*
- **Gate B — one genuinely graph-derived page**: rebuild **page 8** with assertion refs
  + page trace; metamorphic tests (mutate a source object → the page changes or the
  build fails).
- **Gate C — calibrated adjudication**: stratified audit ≥10% per report + high-risk
  oversample; persisted verdicts; per-gate error categories.
- **Gate D — corpus claim defined**: discovery vs validation frames; bounded metrics
  only.
- **Gate E — skeleton re-adjudicated** and re-rendered from a frozen mini-release with
  trace + QA manifests.
- **Gate F — resume verification/merging**, publication-driven (dossier-first), never
  in numerical order.

## Stop conditions (binding)

Stop and escalate when: a page needs an unsupported synthesis to work; a convergence
has no defensible shared proposition; empirical evidence is too indirect for the
intended advice; community authority is unresolved; adjudicators materially diverge on
a keystone; an audit exceeds its predeclared error threshold; an assertion cannot be
traced. **Shipping a beautiful unsupported page is failure; stopping is not.**
